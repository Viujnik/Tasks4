# TaskZ: Асинхронная лихорадка

Добро пожаловать в систему управления задачами, где каждая строка данных проверяется, каждая задача имеет свой характер,
а очередь умеет больше, чем просто хранить элементы. Теперь, очередь, новые обработчики задач и взаимодействие с
задачами - асинхронные!

## Архитектурные слои

### Валидаторы (src/validators)

+ **fields.py** – дескриптор `ValidatedField`, который не пускает грязь в святая святых. Любое присваивание проверяется
  набором правил.
+ **rules.py** – коллекция правил: `TypeRule`, `MinLenRule`, `MaxLenRule`, `MinValueRule`, `MaxValueRule`, `StatusRule`.
  Все они следуют протоколу `Rule` – достаточно реализовать метод `validator(field, value)`.
+ **errors.py** – свои исключения `LenError` и `StatusError`.

### Модель задачи (src/models.py)

Класс `Task` – центральная сущность. Атрибуты защищены дескрипторами:

+ `task_id` (int, положительный)
+ `task_type` (str, 3-16 символов)
+ `description` (str, 8–128 символов)
+ `priority` (int, 1–5)
+ `status` (один из: created, in_progress, in_review, finished)
+ `created_at` (datetime, присваивается автоматически)
+ `deadline` (datetime)
+ `payload` (dict)

**Вычисляемые свойства** (non-data descriptors):

+ `is_on_time` – `True`, если задача завершена (`finished`) и дедлайн не просрочен.
+ `summary` – короткая строка: `Task #123 (type) | status | priority | deadline`

### Источники задач (src/sources.py)

Три способа создать задачу:

+ **FileSource** – имитация файловой системы. Генерирует задачу с рандомным файлом (имя, размер,
  отправитель/получатель).
+ **ConsoleSource** – спрашивает у пользователя команду и случайный статус выполнения (OK/ERROR/WARNING/CRITICAL). С
  помощью yield команда запрашивается только в момент создания.
+ **APISource** – имитация REST API. Метод случайный (GET/POST/PUT/PATCH), URL зашит как `https://rkn.gov.ru`, а статус
  ответа всегда `ERROR` – жизненно.

Каждый источник реализует `get_task() -> Task` и `printf_task(task) -> str` для красивой печати.

### Очередь задач (src/queue.py)

`TaskQueue` внутри использует `Queue` из asyncio, дополнительный реестр `_storage` для генераторов, также для
предотвращения гонки данных используется Lock() из asyncio, и реализует:

+ `async add_task(task)` – добавляет задачу в реестр и очередь с использованием блокировки.
+ `async get_task()` – извлекает задачу из очереди для обработки.
+ `task_done()` - подтверждает выполнение задачи и уменьшает счетчик незавершенных дел.
+ `async join()` - ожидает завершения всех задач, находящихся в очереди.
+ `qsize()` - получает количество задач в очереди.
+ `empty()` - проверяет, пуста ли очередь.
+ `async __aiter__` – получение итератора по текущей копии всех задач в реестре(по копии, так как в момент итерации
  очередь может измениться).
+ `async filter(condition)` – фильтрация задач по копии реестра.
+ `async filter_by_status(status)`, `async filter_by_priority(priority)`, `async filter_by_deadline()` – фильтрация
  задач по полю.
+ `async find_by_id(task_id)` – поиск по ID.
+ `async __aenter__()`, `async __aexit__()` - методы для контекстного менеджера.

Очередь теперь инициализируется в файле main.py.

### Обработчики задач (src/handlers.py)

Все обработчики реализуют единый интерфейс через асинхронный метод `handle(task)`, что позволяет `TasksExecutor`
работать с ними универсально.

* **FileHandler** — имитирует асинхронную работу с файловой системой. Использует `asyncio.sleep` для имитации задержки
  ввода-вывода, не блокируя основной поток.
* **ConsoleHandler** — выполняет системные команды.
    * Использует `asyncio.create_subprocess_exec` для запуска процессов.
    * Парсит аргументы через `shlex.split`, предотвращая Shell Injection.
    * Асинхронно считывает `stdout` и `stderr` через неблокирующие пайпы.
* **APIHandler** — выполняет сетевые HTTP-запросы.
    * Реализован на базе библиотеки `aiohttp`.
    * Использует асинхронные контекстные менеджеры для управления сессиями и соединениями.
    * Получает HTTP-метод (GET, POST и т.д.) через данные в `payload` задачи(или ставит GET, если метод не указан).

## Интерактивная консоль (main.py)

Точка входа в приложение, реализующая асинхронный цикл управления задачами. Основные особенности:

* **Асинхронный интерфейс**: Использование библиотеки `prompt_toolkit` позволяет пользователю взаимодействовать с меню,
  пока фоновые воркеры обрабатывают задачи.
* **Конкурентный вывод**: Применен механизм `patch_stdout`, который предотвращает перемешивание логов от воркеров со
  строкой ввода пользователя.
* **Жизненного цикл**: Запуск и остановка пула воркеров инкапсулированы в асинхронный контекстный
  менеджер `TasksExecutor`.

**Доступные действия:**

1. **Создать задачу(и)** — поддерживает как ручную настройку полей, так и массовую генерацию через источники данных (
   `FileSource`, `ConsoleSource`, `APISource`).
2. **Изменить статус** — поиск задачи в общем реестре по ID и обновление её состояния (дескрипторы модели `Task`
   гарантируют валидность нового статуса).
3. **Очистить N задач** — ручное извлечение задач из очереди (использует `get_task` и `task_done`).
4. **Graceful Shutdown** — при выходе из меню программа дожидается завершения текущих задач в воркерах перед закрытием.


## Тесты

Для тестов используется `pytest`. Все тесты лежат в папке `tests/` и разделены по логике:

+ `conftest.py` – общие фикстуры: данные для задачи и очередь из 3 задач, чтобы не писать одно и то же в каждом файле.
+ `test_task_model.py` – проверка валидации полей (типы, длины, диапазоны, статусы), свойств `is_on_time` и `summary`.
+ `test_queue.py` – тесты очереди: добавление, итерация, фильтрация по условию/статусу/приоритету/дедлайну, поиск по ID,
  извлечение из пустой очереди.
+ `test_sources.py` – проверка, что каждый источник (FileSource, ConsoleSource, APISource) возвращает корректный объект
  `Task`.
+ `test_tasks_functions.py` – Проверка асинхронного взаимодействия функций с очередью для `create_task`, `set_task_status`, `get_left_tasks` (с моками
  ввода, но для простоты без логгера), проверка невозможности изменить статус завершенной задачи.

### Запуск тестов

Из корня проекта (где лежит `pyproject.toml`):

```bash
uv run pytest
```
